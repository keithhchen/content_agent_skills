# 截图 → 结构化数据

## 做什么

读取截图，判断内容类型，输出对应格式：
- **表格类**（有行列结构）→ CSV
- **线性文字**（聊天记录、文章、列表）→ Markdown

## 流程

1. 用 Read 工具读取图片
2. 判断内容类型：表格 or 线性文字
3. 提取内容，输出对应格式
4. 让用户确认，有错误立即修正

## 输出规则

### 表格类 → CSV

- 第一行为表头
- 保持原始列顺序
- 合并单元格的内容填入对应的每一行
- 保存为 `.csv`，`utf-8-sig` 编码

### 线性文字 → Markdown

- 保持原文层级结构
- 聊天记录：`**发送者**：内容` 格式
- 列表：用 `-` 或 `1.` 保持原始编号
- 不添加原文没有的标题或结构

## 原则

- **忠实原文**：不翻译、不改写、不补充。截图里是什么就输出什么
- **不确定的字就标注**：用 `[?]` 标记无法确认的文字
- **多张图片逐张处理**：不要合并不同截图的内容
